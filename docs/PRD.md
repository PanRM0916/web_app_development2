# 讀書筆記本系統 PRD

## 1. 專案概述
### 背景與動機
在資訊爆炸的時代，人們閱讀大量書籍但往往難以保留核心知識。傳統紙本筆記不便搜尋，而一般通用筆記軟體缺乏針對「閱讀流程」的專門設計。本系統旨在建立一個專門的數位環境，讓使用者能從購書、閱讀、記錄到複習，完成一站式的知識內化。

### 目標用戶
- **在校學生**：需整理各學科參考書筆記與重點。
- **職場專業人士**：需快速索引專業書籍中的知識片段。
- **深度閱讀愛好者**：享受系統化收藏閱讀感悟的人群。

### 核心價值主張
「讓每一份閱讀心得都有跡可循，讓知識內化變得更簡單。」

## 2. 功能需求
### 主要功能與使用者故事
1. **書籍清單管理**
   - 作為使用者，我希望可以新增、編輯與刪除我的書籍資訊，以便建立個人圖書庫。
2. **多維度筆記記錄**
   - 作為使用者，我希望在特定書籍下建立筆記，並支援 Markdown 語法，以便美觀地整理重點與想法。
3. **學習進度追蹤**
   - 作為使用者，我希望記錄目前的閱讀頁碼與總頁數，以便視覺化追蹤我的閱讀進度。
4. **標籤與分類過濾**
   - 作為使用者，我希望為筆記或書籍添加自定義標籤，以便在筆記變多時能快速找到相關知識。
5. **複習提醒系統**
   - 作為使用者，我希望系統能根據我的閱讀日期提醒我複習，以便對抗遺忘曲線。

## 3. 非功能需求
### 技術限制
- 後端框架：Flask
- 模板引擎：Jinja2
- 資料庫：SQLite (符合輕量化與單機使用場景)
- 樣式：Vanilla CSS (確保介面純淨且具備高自定義性)

### 效能與安全考量
- 響應式設計：需支援平板與手機閱讀。
- 資料備份：支援匯出筆記為 Markdown 檔案。

## 4. MVP 範圍
